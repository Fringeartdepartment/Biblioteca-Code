# Content OS

Esta pasta é o ponto de entrada editorial do projeto.

Use o Obsidian para criar, revisar e organizar notas em Markdown. O site ainda não lê esta pasta automaticamente; por enquanto ela funciona como fonte editorial versionada para o Codex transformar em código, páginas, produtos, campanhas e novas rotas.

## Estrutura

```txt
content/
  biblioteca/
    produtos/   Obras, assinaturas, prateleiras e fichas de produto.
    paginas/    Páginas de venda, páginas editoriais e páginas institucionais.
    campanhas/  Lançamentos, ofertas, sequências comerciais e anúncios.
  nichos/        Ideias para novos sites, acervos e organismos editoriais.
  templates/     Modelos para copiar no Obsidian.
```

## Status editorial

Use um destes valores no campo `status`:

- `rascunho`: ideia ainda aberta.
- `revisao`: precisa de lapidação de copy, estrutura ou visual.
- `pronto-para-site`: pode ser transformado em página ou seção.
- `publicado`: já entrou no site.
- `arquivado`: não usar por enquanto.

## Regra simples

Cada nota deve responder:

1. O que é isto?
2. Para quem existe?
3. Qual ação o visitante deve tomar?
4. Qual imagem, checkout ou página está ligada a isto?
5. O que precisa preservar da linguagem da Biblioteca dos Mistérios?
